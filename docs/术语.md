# 术语速查

## RCE（Remote Code Execution，远程代码执行）

**攻击者能让你的服务器执行他写的代码。** 安全漏洞里最严重的一档 —— 别的漏洞是偷看一点东西，RCE 是直接把机器交给对方。

怎么来的：程序把**不可信的输入**当代码/命令用了。典型三种：

```js
exec(`ffmpeg -i ${req.query.file}`)   // 用户传 "a.mp4; rm -rf /" → 分号后面照样执行
eval(userInput)                        // 直接把用户输入当 JS 跑
deserialize(userInput)                 // 反序列化能构造出「调用任意函数」的对象
```

**我们的网页版目前没有这类代码**（只有 auth 几个接口 + 周历代理，不执行命令、不反序列化）。但「现在没有」不等于「以后不会有」，所以要有第二道防线 ↓

## 权限隔离 / 最小权限原则

**假设第一道防线迟早会破，让破了之后的损失尽量小。**

网页版后端原来用 **root** 跑，等于「万一有 RCE，攻击者直接就是 root」—— 读所有文件、装后门、删备份、改 nginx 劫持流量。2026-07-17 改成用专用低权限用户 `mapletools` 跑：

![权限隔离前后对比](devlog-assets/least-privilege.svg)

同一个漏洞，损失从「整台机器」缩到「网页版自己的那点数据」。**备份目录故意留给 root**，应用用户读不到也删不掉 —— 勒索软件的标准操作就是先毁备份再加密。

## 纵深防御（Defense in Depth）

**别指望任何单一措施不出错，一层破了还有下一层。** 网页版现在的层：

| 层 | 挡什么 |
|---|---|
| node 只绑 `127.0.0.1` | 绕开 nginx 直连后端 |
| nginx + HTTPS + HSTS | 明文窃听、中间人 |
| 登录/注册/找回**限流** | 暴力猜密码 |
| scrypt 哈希 | 库被拖走后密码被反推 |
| `token_version` | 改密码后老会话还能用 |
| **低权限用户跑** | RCE 之后的横向扩大 ← 本节 |
| 备份 root-only + 每天快照 | 数据被毁 / 误删 |
