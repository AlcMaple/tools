## 📖 详细说明

### 封面本地化（显示层解析，非持久化）

**目标效果**：所有封面（MyAnime 列表 / Calendar 网格 / AnimeInfo 详情）走
本地文件，离线可看，不每次重新拉 lain.bgm.tv；老追番渲染时自动回填。

**已经做过的事**（避免重新踩坑）：

- **封面 ≠ 限流原因**：封面走 `lain.bgm.tv` 图片 CDN，跟 `api.bgm.tv`（搜索/
  详情/周历，限流就在这）是**两套独立限流账本**。本地化的真正价值是
  性能 + 离线，**不是**为了缓解限流。所以 `cover-cache.ts` 不接 RateLimiter
- **⚠️ 踩坑：本地路径不能持久化进 track.cover**。第一版用 `cacheCoverFor`
  把下载后的 `archivist:///Users/macA/.../covers/123.jpg` 写进 track.cover 并
  落盘。但 track.cover 要跨设备 WebDAV 同步 —— 这个路径含**本机 userData
  绝对路径**，同步到设备 B 后路径根本不存在 → 封面全裂。**已废弃这个做法**
- **正确做法**：track.cover **永远存可移植的 URL**（同步安全），本地化只在
  **显示时**由 `hooks/useCover.ts` 按设备各自做 —— 每台机器下到自己的
  userData/covers/，互不干扰
- **useCover 统一三处**（MyAnime / Calendar / AnimeInfo），一套机制：
  - 渲染时拿 URL → 查本地有没有 → 有就 archivist://，没有后台下载完再切
  - 老 40 部追番的"回填"是这个机制的**自然副作用**（渲染就触发下载），
    不用单独写回填逻辑
  - 模块级 Map 缓存已解析路径，避免重复 IPC；失败保留 URL 不重试
- **archivist:// 已有协议**（main/index.ts:96）服务任意本地文件，封面存
  `userData/covers/{key}.{ext}` 直接复用，不用新协议
- **不做"2 周覆盖"**：封面图基本永不变（变的是番剧阵容不是封面图），
  cover-cache skip-if-exists 即可。真要刷新以后加 force 参数

### 手动添加

**目标效果**：限流时不靠 BGM 搜索也能加番，且填了真 bgmId 后限流恢复
无缝衔接（详情页认出已追番 + 自动补全）。

**已经做过的事**（避免重新踩坑）：

- **bgmId 设计**：用户优先填真的（从 bgm.tv 条目 URL 拿，限流不影响浏览器
  访问 bgm.tv 主站）→ 用真 id。填不出来 → 留空 → 系统给 `-Date.now()`
  负数兜底（真 BGM id 都是正整数，亲不冲突）。用户决策见 006 起始
  AskUserQuestion
- **为什么填真 id 就无缝**：track store（maple-anime-tracks-v1）和 detail
  cache（search_cache.json 的 bgm_detail_cache）**物理隔离**。手动加只写
  track store 的最少字段，detail cache 那边是空的。详情页 `useAnimeTrack(bgmId)`
  按 bgmId 查 store → 命中手动 track → 显示已追番；`getCachedBgmDetail(bgmId)`
  miss → fresh fetch → 拿全量。手动加的最少数据**不会污染** detail cache
- **入口位置**：MyAnime 各类目 tab 顶部（动画/漫画/小说），不是 AnimeInfo
  限流出错屏。限流不限流都能用，不是限流专用。用户决策见 AskUserQuestion
- **最少必填**：标题 + 类目（默认 = 当前 tab）。bgmId / 中文标题 / 封面 /
  总集数都可选。状态默认 plan，集数 0
- **重复 bgmId 拦截**：填的真 bgmId 已在追番列表里 → 报错不让加（避免覆盖）
