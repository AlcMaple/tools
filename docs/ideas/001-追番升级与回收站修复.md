**目标效果**：把刚写完的功能动手走一遍，确认没回归。覆盖范围：

1. **追番记录**

   - SearchDownload 卡片角「关联追番」按钮 → mini BGM 搜索弹窗 → 选条目
     → 卡片左上角 EP 徽章出现
   - MyAnime 汇总页：状态 segment / −/数字/+1 集数 / 来源 chip
   - MyAnime「+ 添加链接」按钮 → B 站 preset / 自定义 → URL 必填 + 去重
   - WebDAV 同步：Push 后到 HomeworkLookup 的对比弹窗看「追番 N 部 / 在追 N」

2. **番剧周历**
   - /calendar 入口 → 7 列网格自适应（默认 1280px 窗口不溢出）
   - 当日列高亮，无 TODAY 字样
   - 滚动后置顶是周一-周日 chip + 缓存/刷新按钮，不是页面标题
   - chip 列和卡片列像素级对齐
   - 卡片高度齐平（标题强制 2 行）
   - 刷新按钮强制 update=true 绕过 14 天缓存

3. **跳转观看**

   - Aowu chip 点击跳 `aowu.tv/w/{token}` 用户页（不是 /v/{id} 报错页）
   - 老 binding（绑过 Aowu 但还是 /v/{id} 形式）首次渲染 MyAnime 时

   - Xifan / Girigiri / Bilibili / Custom chip 点击直接打开 sourceUrl

**已经做过的事**（避免重新踩坑）：

  browser fingerprint + 限流页检测；不要往下压 timing 否则会被 30s 关

  数据，但不要再加备注 UI；用户明确说不要

### File Explorer 删除：PowerShell 结果解析失败

**目标效果**：Windows 下删除被占用 / 系统保护的文件夹时：
- 能删的——杀进程、takeown、icacls 等自动尝试，最后 SendToRecycleBin
  或 Remove-Item 成功，toast 显示「自动关闭 N 个占用程序」
- 删不掉的——弹窗「原始错误」段是真实可读的失败原因（"Access is
  denied" / "Cannot find path" 之类 Windows 原文），而不是当前的
  `SendToRecycleBin: PowerShell 结果解析失败` 白板字串

**复现**：Windows 下选 `C:\Users\<user>\Downloads\maple-tools-20260511-1748`
（应用自身的安装/解压目录，正在被 Maple Tools 进程占用），点删除：
- 「移到回收站」失败弹窗 原始错误 = `SendToRecycleBin: PowerShell 结果解析失败`
- 「永久删除」失败弹窗 原始错误 = `Remove-Item: PowerShell 结果解析失败`

**已经做过的事**（避免重新踩坑）：
- 3ad3e9e 已经把删除统一成「单一作用」模型：trash() / permanentDelete()
  内部自动 kill 进程 + takeown + icacls，UI 上只有这两个顶层按钮，没
  「强制删除」/「升级永久删除」之类的中间态。这次修 bug 不要回到给用户
  开第三个按钮的老路
- PS 子进程改成始终 exit 0，结果通过 `MAPLE_RESULT_BEGIN/END` marker
  包 JSON 透传给主进程；TS 侧 `parsePsDeleteResult` 找不到 marker 时
  兜底文案就是当前看到的"结果解析失败"。所以这次要做的是让 TS 侧把
  PS 真正的 stdout / stderr 透传到弹窗，而不是改 PS 输出格式
- 用户明确否过自动 fallback（弹窗里给「改为永久删除」按钮），所以失败
  弹窗只能展示信息，不能提供"试试更激进的方式"入口
