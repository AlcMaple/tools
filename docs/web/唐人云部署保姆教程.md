# 网页版部署（唐人云 VPS · 正在跑的这套）

线上就是这套：`https://anime.alcmaple.cn`。
另外两份是备选，别搞混：`Vercel部署保姆教程.md`（免费但国内慢、SQLite 存不住）、`Oracle开机保姆教程.md`（甲骨文那台）。

## 现状速览

| 东西 | 在哪 | 备注 |
|---|---|---|
| 代码 | `/opt/mapletools`（git clone，`main` 分支） | 发布 = 在这里 `git pull` |
| 跑的目录 | `/opt/mapletools/web` | pm2 的 `cwd` |
| **数据 + 密钥** | `/opt/mapletools-data/` | **故意在仓库外**，见下方红线 |
| 备份 | `/opt/mapletools-backup/` | 每天 04:00，留 14 天 |
| 进程 | pm2 `mapletools-web` → `npm start` → `server/node.ts` | **跑在 `mapletools` 用户名下，不是 root**。开机自启 `pm2-mapletools` |
| 入口 | nginx → `127.0.0.1:3000` | 证书 certbot 自动续 |
| 机器 | Ubuntu 22.04 · 2G 内存 · node 22（nodesource） | `ssh -p 31789 root@103.250.172.17` |

### 红线：这三条踩了就出事

1. **`DATA_DIR` 必须在部署目录外。** 用户库是 `/opt/mapletools-data/web.db`。早期方案是 `rm -rf /opt/web` 再传新的 —— 那会把所有用户一起删掉。现在改成 git pull 就不会了，但数据仍然留在外面：换回打包部署时不会有人记得这茬。
2. **`AUTH_SECRET` 不进仓库。** 它在 `/opt/mapletools-data/ecosystem.config.cjs` 里。换掉它 = 所有人被登出（JWT 签名全废）。
3. **node 只绑 `127.0.0.1`。** nginx 是唯一入口。绑 `0.0.0.0` 的话公网直连 `http://<ip>:3000` 就绕开 nginx，登录密码明文过网，且 `X-Forwarded-For` 随便伪造、限流形同虚设（真踩过，2026-07-17 修）。
4. **应用不用 root 跑。** 跑在 `mapletools` 用户名下 —— 万一后端出 RCE，攻击者只能拿到这个用户的权限（读写网页版自己的库），碰不到系统、SSH 密钥、也删不掉备份。概念见 [`docs/术语.md`](../术语.md)。

## 日常发布（99% 的时候只用这个）

本地 push 之后：

```bash
ssh -p 31789 root@103.250.172.17
cd /opt/mapletools && git pull
cd web && npm ci && npm run build
mtweb restart mapletools-web --update-env
mtweb logs mapletools-web --lines 20   # 看一眼没报错
```

只改了前端就不用 `--update-env`；改了 `ecosystem.config.cjs` 里的环境变量才需要。

> 这套**不会更新番剧索引** `bgm_index.db`（它不在 git 里）—— 那个走 cron / 手跑同步脚本，见下面「番剧索引」那节。

> **`mtweb` 是什么？为什么不是 `pm2`？**
> 应用跑在 `mapletools` 用户名下，pm2 守护进程是**按用户各一份的** —— root 直接敲 `pm2 list` 会是**空的**（别慌，站点没挂）。
> `mtweb`（`/usr/local/bin/mtweb`）就是 `su mapletools + cd + pm2 $*` 的包装，用法跟 pm2 完全一样。
> 那个 `cd` 是必须的：少了它 pm2 会继承 root 的 `/root`（700）当工作目录，然后报一个**指向 node 的假错误** `spawn /usr/bin/node EACCES` —— 查半天以为是 node 权限问题，其实是 cwd。包装器就是为了把这个焊死。

**验一下真上去了**（别只看 pm2 说 online）：

```bash
curl -s https://anime.alcmaple.cn/api/auth/questions | head -c 80
```

> ⚠️ **2G 内存 + 没有 swap**，`npm run build` 有 OOM 的可能。构建被 `Killed` 就是这个。临时开 swap：
> ```bash
> fallocate -l 2G /swapfile && chmod 600 /swapfile && mkswap /swapfile && swapon /swapfile
> ```

## 番剧索引 `bgm_index.db`（搜索加番的数据源）

追番页「加番」搜的是本地库 `/opt/mapletools-data/bgm_index.db`（BGM 官方离线档里的全部动画，~3 万条、5.4MB）。

**它不在 git 里、也不是 `npm run build` 的产物** —— 所以「日常发布」那一套 `git pull && npm ci && npm run build` **不会**更新它，甚至新机器上压根没有这个文件。它只有一条来源：跑同步脚本。

```bash
apt install -y unzip                     # 一次性前置：脚本靠 unzip 流式读压缩包
su -s /bin/bash mapletools -c 'cd /opt/mapletools/web && DATA_DIR=/opt/mapletools-data npm run sync:index'
```

- **首次部署必须手跑一次**，否则加番搜索一直是「索引还没生成」。
- 用 `mapletools` 跑（不是 root）：数据目录是它的，root 跑会留下 root 属主的文件。`DATA_DIR` 必须显式给 —— 那是 pm2 配置里的变量，cron/手敲的 shell 不会继承。
- 干的事：下 ~419MB zip 到 `/tmp` → `unzip -p` 流式过一遍 → 只留 type=2 → 写 `bgm_index.db.tmp` → **rename 原子替换** → 删 zip。跑完 `/tmp` 不留东西，但跑之前得有 1G 空档（`df -h /tmp`）。
- **不用重启 pm2**。服务端按 mtime 判断索引换没换，下一次搜索自动读新库（`server/bgm/anime-index.ts`）。
- **不占内存**（流式 + 分批入库），跟 `npm run build` 那个 OOM 风险不是一回事。

**每周自动更新**：上游 `bangumi/Archive` 每周三出新档，所以挂周四凌晨（留一天余量）：

```bash
crontab -l 2>/dev/null | { cat; echo '0 5 * * 4 su -s /bin/bash mapletools -c "cd /opt/mapletools/web && DATA_DIR=/opt/mapletools-data npm run sync:index" >> /var/log/mapletools-index.log 2>&1'; } | crontab -
```

**验证同步到哪天了**（`q` 传空就只回统计，不搜东西）：

```bash
curl -s 'https://anime.alcmaple.cn/api/search?q=' 
# {"ready":true,"total":30502,"builtAt":1753171200000,"data":[]}
```

`ready:false` = 没跑过脚本或文件丢了；`total` 明显偏小 = 上次跑到一半（不会发生，原子替换保证要么全旧要么全新，真见到就是档本身出问题了）。

**不进备份**：`mapletools-backup.sh` 只备 `web.db`，这库是可再生的（重跑脚本就回来了），没必要每天塞 5MB 进备份盘。

> **时效**：跟着周档走，所以本周新建的 BGM 条目最迟下周四才进索引。冷门/极新番搜不到就是这个原因。

## 换新机器：从零搭

```bash
# 1. node 22 + pm2 + nginx + certbot
curl -fsSL https://deb.nodesource.com/setup_22.x | bash - && apt install -y nodejs nginx sqlite3 unzip
npm i -g pm2

# 2. 代码
git clone https://github.com/AlcMaple/tools.git /opt/mapletools
cd /opt/mapletools/web && npm ci && npm run build

# 3. 专用低权限用户（应用不能用 root 跑，见红线 4）
#    --system = 系统账号、密码位锁死，没人能拿它登录
useradd --system --create-home --home-dir /home/mapletools --shell /bin/bash mapletools

# 4. 数据目录（仓库外！）交给它；备份目录**故意留给 root**，应用被攻破也删不掉备份
mkdir -p /opt/mapletools-data && chown mapletools:mapletools /opt/mapletools-data
chmod 700 /opt/mapletools-data
mkdir -p /opt/mapletools-backup && chmod 700 /opt/mapletools-backup   # root:root
```

`/opt/mapletools-data/ecosystem.config.cjs`：

```js
module.exports = {
  apps: [{
    name: 'mapletools-web',
    script: 'npm',
    args: 'start',
    cwd: '/opt/mapletools/web',
    env: {
      NODE_ENV: 'production',      // 少了它 cookie 不带 Secure
      PORT: '3000',
      DATA_DIR: '/opt/mapletools-data',
      AUTH_SECRET: '<openssl rand -hex 32 生成，别用示例值>'
    }
  }]
}
```

`ecosystem.config.cjs` 也要给 `mapletools`（它得读到 `AUTH_SECRET`）：`chown mapletools:mapletools /opt/mapletools-data/ecosystem.config.cjs`

```bash
# 5. mtweb 包装器（把 su + cd 焊死，见「日常发布」那节的解释）
cat > /usr/local/bin/mtweb <<'EOF'
#!/bin/bash
set -e
[ "$(id -u)" = 0 ] || { echo "要用 root 跑"; exit 1; }
exec su -s /bin/bash mapletools -c "cd /opt/mapletools/web && pm2 $*"
EOF
chmod 755 /usr/local/bin/mtweb

# 6. 起进程 + 开机自启（都在 mapletools 名下）
mtweb start /opt/mapletools-data/ecosystem.config.cjs
pm2 startup systemd -u mapletools --hp /home/mapletools   # 这条用 root 跑，生成 pm2-mapletools 单元
mtweb save                  # 少了这步，重启后站点不会自己回来

# 7. 番剧索引：新机器上这文件不存在，不跑加番搜索就是空的（详见上一节，含每周 cron）
su -s /bin/bash mapletools -c 'cd /opt/mapletools/web && DATA_DIR=/opt/mapletools-data npm run sync:index'

# 8. nginx（先 http，certbot 会自己改成 https）
```

> **验开机自启别偷懒**：直接 `systemctl restart pm2-mapletools` 会**假失败**（守护进程已在跑 → `pm2 resurrect` 无需 fork → systemd 报 `Result: protocol`）。要先 `mtweb kill` 把守护进程杀干净，再 `systemctl start pm2-mapletools`，那才是真开机的条件。

`/etc/nginx/sites-available/mapletools-web`：

```nginx
server {
    listen 80;
    server_name anime.alcmaple.cn;
    client_max_body_size 20m;

    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;          # 限流靠它，别删
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

> `add_header` 放 server 级；**一旦在某个 `location` 里再写 `add_header`，继承来的会被整组顶掉**，得在那个 location 里重写一遍。

```bash
ln -s /etc/nginx/sites-available/mapletools-web /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx

# 6. HTTPS（DNS 先解析过来，只留一条 A 记录指向本机）
certbot --nginx -d anime.alcmaple.cn --redirect
systemctl list-timers | grep certbot   # 确认自动续期的 timer 在

# 7. 备份（详见下节）
```

## 系统更新

安全补丁由 `unattended-upgrades` 自动打，平时不用管。MOTD 上「XXX updates can be applied」这个数**看着吓人但基本是噪音** —— 绝大多数是 `-updates` 里的普通 bug 修复，不是安全更新。真要看有几个安全更新：

```bash
apt-get -s upgrade | grep "^Inst" | grep -c security
```

**内核**（`linux-image-*`）会被 `apt upgrade` **held back**，得 `apt full-upgrade` 才升，且**必须重启才生效**。

> **`cloud-init` 已 `apt-mark hold` 冻结**（`apt-mark showhold` 能看到）。系统镜像本来就把它排除在自动更新之外；VPS 上升级 cloud-init + 重启是经典的「网络配置被重新生成、机器起不来」场景，而这台机根本不需要它的新特性。**要解冻先想清楚为什么。**

重启前后：`pm2 save`（快照要是旧的，重启后恢复的就是旧配置）→ `reboot` → 等 1 分钟站点自己回来（nginx 起得比 node 快，中间那几秒是 502，正常）。

## 备份

`/usr/local/bin/mapletools-backup.sh`，crontab `0 4 * * *`，**以 root 跑**（这样备份目录能保持 root-only，应用被攻破也删不掉备份）：

```bash
sqlite3 /opt/mapletools-data/web.db ".backup /opt/mapletools-backup/web-$(date +%F).db"
gzip -f /opt/mapletools-backup/web-$(date +%F).db
find /opt/mapletools-backup -name "web-*.db.gz" -mtime +14 -delete
chown mapletools:mapletools /opt/mapletools-data/web.db{,-wal,-shm}   # ← 别删这行，见下
```

- **必须用 `sqlite3 .backup`，不能 `cp`** —— 库开着 WAL，`cp` 会在写入中途拷到一个撕裂的文件。
- **最后那行 `chown` 是必须的**：root 打开库时若 WAL/SHM 不存在会**新建成 root 所有**，应用（mapletools）就再也写不了自己的库 —— 会在某个凌晨 4 点之后毫无征兆地 500。
- **保留 14 天 ≠ 只存 14 天的数据**。每份都是**全库完整快照**（不是增量），今天那份含全部历史数据。删掉老快照不丢数据，丢的只是「回滚到 14 天前某一刻」的能力。库才几 KB，想留更久把 `-mtime +14` 改大即可。

恢复：

```bash
mtweb stop mapletools-web
gunzip -c /opt/mapletools-backup/web-<时间戳>.db.gz > /opt/mapletools-data/web.db
rm -f /opt/mapletools-data/web.db-wal /opt/mapletools-data/web.db-shm   # 旧 WAL 会盖掉恢复的内容
chown mapletools:mapletools /opt/mapletools-data/web.db && chmod 600 /opt/mapletools-data/web.db
mtweb start mapletools-web
```

## 排错

| 现象 | 多半是 |
|---|---|
| **`pm2 list` 是空的** | 你在用 root 敲。应用在 `mapletools` 名下，pm2 守护进程按用户各一份 → 用 `mtweb list` |
| **`spawn /usr/bin/node EACCES`** | **不是 node 的权限问题**（`su -s /bin/bash mapletools -c "node -v"` 明明能跑）。是 `su` 不切工作目录、继承了 root 的 `/root`（700）→ 加 `cd` 或直接用 `mtweb` |
| 502 Bad Gateway | node 挂了 → `mtweb logs mapletools-web`；或没绑 3000 → `ss -tlnp \| grep 3000` |
| 站点 500、日志说数据库 readonly | WAL/SHM 被 root 拿走了属主 → `chown mapletools:mapletools /opt/mapletools-data/web.db{,-wal,-shm}`（备份脚本最后那行就是防这个） |
| `/api/auth` 一打就 500 | `better-sqlite3` 是原生模块，换过 node 大版本要 `npm rebuild better-sqlite3` |
| 登录成功但刷新就掉线 | cookie 没带 `Secure`/没写进去 → 查 `NODE_ENV=production` 有没有生效（`mtweb restart mapletools-web --update-env`） |
| 改完代码没生效 | `npm run build` 没跑，或跑在了别的目录 → `mtweb describe mapletools-web \| grep cwd` |
| 构建被 `Killed` | 2G 内存 OOM，开 swap（见上） |
| 证书过期 | `certbot renew --dry-run` 看报错；HSTS 是 1 年，证书一过期浏览器会**硬拒绝**、点不进去 |

**排查线上问题时，状态码没有意义** —— SPA 兜底让任何路径都回 200 + `index.html`。`curl https://anime.alcmaple.cn/.git/config` 返回 200 不代表泄露了，得看 body 是不是那 440 字节的 `index.html`。
