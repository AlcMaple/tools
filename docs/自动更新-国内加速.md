# 自动更新 —— 国内加速 + GitHub 回退

> 代码：`src/main/updater/index.ts`、`update-manifest.json`、`scripts/sync-manifest.mjs`、
> 设置项 `updateSource`（`src/main/ipc/system.ts` + Settings 页「更新源」）。

## 为什么是现在这套

GitHub Release 产物走 `objects.githubusercontent.com`，国内无魔法直接拉不到，连 feed 文件
`latest.yml` 都下不来 → 更新流程死在第一步（早期走 Cloudflare 同样栽在这）。

实测 ghproxy 系镜像（`https://ghproxy.net/` 前缀反代 GitHub）国内可达，但有两个限制：

- ❌ 不认 `/releases/latest/` 重定向（502）、不放行 `releases.atom`（403）
- ✅ 固定 tag 的产物 `{proxy}https://github.com/owner/repo/releases/download/vX/...` 能下
- ✅ `raw.githubusercontent.com` 上的小文件（经 ghproxy / jsdelivr）能读

所以不能让更新器「自己找最新版」，必须**两步走**。

## 两步走

1. **查版本**：读 repo 根目录的 `update-manifest.json`（`{ version, proxies }`）。
   通道带回退：ghproxy-raw → jsdelivr → 直连 raw。
2. **下载安装**：拿到版本号后拼**固定 tag** 的 ghproxy URL，交给 electron-updater 的
   **generic provider**（`setFeedURL` + `checkForUpdates` + `downloadUpdate`）逐个代理尝试，
   任一成功即止，全挂回退直连 GitHub。复用 electron-updater 的 NSIS 下载 / 安装。

> 关键验证（electron-updater 6.8.3）：generic provider 内部用原生 `new URL(path, base)` 拼
> 文件 URL，嵌套的 `https://` 会被完整保留（`//` 不被吞），所以 ghproxy 前缀 base 可用。

平台差异：

- **Windows**：generic provider 真·静默下载 + 重启安装。
- **macOS**：未签名 / 公证，quitAndInstall 在 Sequoia 后静默失败，所以只「查版本 → 给一个
  ghproxy 加速的 dmg 直链」，用户在浏览器下完自行拖进 Applications。

## proxies 列表：可远程更新，不用发版

`update-manifest.json` 的 `proxies` 是「下载阶段」用的代理前缀列表。**某个代理挂了 / 变慢，
直接改这文件 push 到 main，所有已安装客户端下次检查就拿到新列表，无需重新发版。** 这是这套
方案应对「ghproxy 域名经常轮换」的关键。

> 注意 `src/main/updater/index.ts` 里还写死了一两个 `BOOTSTRAP_PROXIES`，那是「拉 manifest
> 本身」用的引导代理（先有鸡还是先有蛋）。它有 jsdelivr + 直连兜底，但如果这一两个引导代理
> 长期全挂，就得改代码 + 发版了——所以引导代理选当下最稳的。

## 更新源设置（updateSource）

- `auto`（默认）：代理链在前、GitHub 直连兜底。覆盖无魔法用户。
- `github`：强制直连 GitHub，跳过所有代理。给有魔法 / 有代理的用户。

## 发版 checklist（和 11.3 配合）

```
□ bump package.json 版本号
□ npm run sync:manifest          # 同步 update-manifest.json 的 version
□ 二者随 release commit 一起提交
□ push v* tag → Actions 打包 draft → 手动发布 draft（资产公开后 ghproxy 才能下）
□ 上线前在「真·无魔法国内网络」curl 一下：
    update-manifest.json（经 ghproxy-raw）能拿到 ✓
    {proxy}.../releases/download/vX/latest.yml 能拿到 ✓
  —— 拉不到就别发，这是上次 CF 翻车的教训。
```
