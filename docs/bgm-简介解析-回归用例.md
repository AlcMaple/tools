# BGM 简介解析 — 回归用例

`src/main/bgm/detail.ts` 里 `extractChineseSummary()` 这块逻辑历来是"按下葫芦浮起瓢"的雷区——BGM 上的简介格式非常杂，中日并排、纯中文、纯日文原文、带 marker、不带 marker 都有；每次改判定/切分规则都容易踩到别的场景。

本文件维护**所有已知踩过坑的测试样本**。每次修改简介相关逻辑（`detail.ts` 的 `extractChineseSummary` / moegirl fallback / `AnimeInfo.tsx` 的简介渲染）之前 **必须** 把下面的每个样本都打开过一遍确认没回归。

新发现 bug → 在「修改历史」追一笔 commit 引用 + 把当时踩坑的番加进「测试用例」列表，以后就在回归集里了。

## 测试用例

### 噬血狂袭 IV
- **BGM**: https://bgm.tv/subject/292073
- **场景**: BGM 上**没有中文简介**，简介字段挂的是纯日文原文，且原文里没有 `[简介原文]` 这种 marker
- **moegirl fallback**: 也找不到 / 找到的也不是好用的中文版
- **预期**: 完整显示日文原文（多段、多行结构保持），不能被切分算法撕成几个碎句
- **历史踩坑**: 早期 `extractChineseSummary` 按 `\n` 逐段判假名密度过滤，短句段（如「物語の舞台は魔族特区"恩莱島"」假名只 2 个）被误判成中文留下，长句段被过滤掉，15 行简介剩 3 行（见 commit `058bde0`）

### Yes! 光之美少女 5 GoGo!（剧场版）
- **BGM**: https://bgm.tv/subject/4419
- **场景**: BGM 上**中文简介 + 日文原文直接拼在一起**，中间只有空格，**没有** `[简介原文]` marker。中文部分 ~130 字，日文部分 ~500 字
- **预期**: 只显示中文简介那一段，结尾必须**完整**含 "...决战镜之国。"，日文部分不要拼上去
- **历史踩坑**:
  - **第一坑**：整段假名密度判定 → 全文 `hasChinese=false` → fallback 走 moegirl 找不到合适中文版 → 直接显示原始字段（中日文混着给用户）
  - **第二坑**（修第一坑顺手踩的）：用 `findJapaneseRegionStart` 滑动窗口返回的"窗口起点"当切点，但窗口起点是为了凑齐 6 个假名往后看 30 字得到的，**起点本身往往还在中文段内** —— 用户看到中文段被砍掉尾巴 ~22 字（"...勇闯镜之" 后面的内容全没了）。修法：拿到 jpStart 后还要在窗口里找**第一个假名字符**的精确位置，那才是真实边界

## 修改历史

> 每次动 `extractChineseSummary` / 简介相关渲染都在这里追一笔。

- **2026-05-15** `058bde0` —— 修复纯日文简介被逐段过滤撕成碎句的 bug。改成：检测到 marker → 取前半中文段；没 marker → 整段原样返回，用整段假名密度判 `hasChinese`。从此再也不"撕段"。
  - 触发样本：噬血狂袭 IV
- **2026-05-18** —— 中文 + 日文无 marker 拼接的混合场景。新增 `findJapaneseRegionStart` 滑动窗口探测，窗口大小 30 / 阈值 6 假名（20% 密度）。**边界 > 0** 才切，边界 = 0 退回老逻辑（这条约束守住"噬血狂袭 IV 不撕段"）。切完的中文段额外要求 ≥5 个汉字防御零散字符误切。
  - 触发样本：Yes! 光之美少女 5 GoGo!（BGM 4419）
  - 不回归验证：噬血狂袭 IV（边界 = 0 走老路径，行为完全不变）
- **2026-05-18** —— 修上一坑：`findJapaneseRegionStart` 返回的"窗口起点"被直接拿来当切点，但窗口起点为了凑 6 个假名往后看了 30 字，**起点本身还在中文段内**。结果中文段被砍掉尾巴 ~22 字（4419 案例"决战镜之国。"丢成"勇闯镜之"）。修法：拿到 jpStart 后**在窗口里找第一个假名字符位置**，用它当真实切点。
  - 触发样本：Yes! 光之美少女 5 GoGo!（BGM 4419）—— 中文末尾必须**完整**到"决战镜之国。"，少一个字都不算过
