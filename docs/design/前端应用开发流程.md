# Archivist 前端应用 0→1 开发流程

## 一、技术选型背景

原有工具链为纯 Python 命令行脚本，运行在 `tools.bat` 菜单中。GUI 化目标：

- 保留后端 Python 脚本逻辑不动
- 用现代 Web 技术栈构建桌面 UI，视觉自由度最高
- 选型：**Electron + React + TypeScript**，使用 **electron-vite** 作为构建工具

> 选择 Electron 而非 PyQt 的原因：CSS/Tailwind 对于复杂暗色设计系统的表现力远优于 Qt 样式表；团队对 Web 技术栈更熟悉。代价是打包体积更大，IPC 通信需要跨进程。

## 三、构建配置

### 3.1 electron.vite.config.ts

三进程独立配置的关键：

```ts
import { resolve } from 'path'
import { defineConfig, externalizeDepsPlugin } from 'electron-vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  main: {
    plugins: [externalizeDepsPlugin()]   // Node.js/Electron 模块不打包
  },
  preload: {
    plugins: [externalizeDepsPlugin()]
  },
  renderer: {
    resolve: {
      alias: { '@renderer': resolve('src/renderer/src') }
    },
    plugins: [react()]
  }
})
```

`externalizeDepsPlugin` 确保 `electron`、`path` 等 Node 内置模块不被 Vite 打包进 bundle，而是在运行时由 Electron 提供。

### 3.2 TypeScript 多配置方案

Electron 项目需要两套 TS 配置：

- **`tsconfig.node.json`**（main + preload）：目标 Node 环境，需要 `"types": ["node"]` 才能识别 `__dirname`、`process` 等全局变量
- **`tsconfig.web.json`**（renderer）：目标浏览器环境，使用 `jsx: react-jsx`，不需要 Node 类型

根 `tsconfig.json` 用 `references` 将两者串联，让 IDE 能按文件位置自动选择正确配置。

**常见陷阱：** 如果忘记在 `tsconfig.node.json` 中加 `"types": ["node"]`，主进程中所有 `__dirname` 和 `process.env` 调用都会报 TS 错误。

## 四、Electron 主进程

### 4.1 src/main/index.ts

主进程负责创建 BrowserWindow，核心逻辑：

```ts
function createWindow(): void {
  const mainWindow = new BrowserWindow({
    width: 1280,
    height: 800,
    show: false,               // 等待 ready-to-show 再显示，避免白屏闪烁
    autoHideMenuBar: true,
    webPreferences: {
      preload: join(__dirname, '../preload/index.js'),
      sandbox: false           // 允许 preload 访问 Node API
    }
  })

  mainWindow.on('ready-to-show', () => mainWindow.show())

  // 阻止在新窗口打开链接，改用系统浏览器
  mainWindow.webContents.setWindowOpenHandler((details: { url: string }) => {
    shell.openExternal(details.url)
    return { action: 'deny' }
  })

  // 开发模式加载 Vite dev server，生产模式加载本地 HTML
  if (process.env['ELECTRON_RENDERER_URL']) {
    mainWindow.loadURL(process.env['ELECTRON_RENDERER_URL'])
  } else {
    mainWindow.loadFile(join(__dirname, '../renderer/index.html'))
  }
}
```

### 4.2 src/preload/index.ts

通过 `contextBridge` 向渲染进程暴露安全 API：

```ts
import { contextBridge } from 'electron'

contextBridge.exposeInMainWorld('electron', {
  versions: {
    node: process.versions.node,
    chrome: process.versions.chrome,
    electron: process.versions.electron,
  }
})
```

当前仅暴露版本信息。后续对接后端 Python 时，在此处添加 `ipcRenderer.invoke` 封装。

## 五、渲染进程（React 应用）

### 5.1 入口：index.html

```html
<!-- CDN 加载字体和图标（Electron 无网络限制） -->
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&family=Space+Grotesk:wght@400;500;600;700&display=swap" rel="stylesheet">

```

Material Symbols Outlined 图标使用方式：`<span className="material-symbols-outlined">icon_name</span>`

### 5.2 路由架构：App.tsx

Electron 中**必须使用 HashRouter**（不是 BrowserRouter），因为没有 HTTP 服务器处理 HTML5 History API 的路径请求：

```tsx
import { HashRouter, Routes, Route } from 'react-router-dom'

function App() {
  return (
    <HashRouter>
      <div className="flex bg-surface min-h-screen">
        <Sidebar />
        <main className="ml-64 pt-16 flex-1">
          <Routes>
            <Route path="/" element={<SearchDownload />} />
            <Route path="/queue" element={<DownloadQueue />} />
            <Route path="/anime-info" element={<AnimeInfo />} />
            <Route path="/biu-sync" element={<BiuSync />} />
          </Routes>
        </main>
      </div>
    </HashRouter>
  )
}
```

### 5.3 布局系统

全局布局为两列：

- **Sidebar**：固定宽度 256px，`position: fixed`，左侧
- **TopBar**：固定高度 64px，`position: fixed`，`left: 256px`，顶部
- **main**：`margin-left: 256px`，`padding-top: 64px`，滚动内容区

TopBar 接受 `rightSlot` prop，允许各页面注入页面特定内容（如下载队列页的速度指示器）。

### 5.4 Sidebar 导航

使用 `NavLink` 实现活跃状态自动管理：

```tsx
<NavLink
  to="/queue"
  className={({ isActive }) =>
    isActive
      ? 'flex items-center gap-3 px-4 py-3 text-primary font-bold border-r-2 border-primary bg-surface-container'
      : 'flex items-center gap-3 px-4 py-3 text-on-surface-variant hover:bg-surface-container'
  }
>
```

## 六、页面实现

### 6.1 搜索下载页（SearchDownload）

**关键实现点：**

- 搜索栏：文本输入 + 线路下拉选择（Xifan / Girigiri / Nyaa.si / AnimeBytes）+ INITIALIZE 按钮
- 结果网格：`grid-cols-4`，每张卡片固定 `aspect-[2/3]`，hover 时浮现 "SELECT ENTRY" 覆盖层
- **验证码弹窗**：`useState(true)` 初始可见，通过 X 按钮关闭。弹窗使用 `glass-effect` 毛玻璃样式（`backdrop-blur-xl bg-white/5 border border-white/10`）

```tsx
const [showCaptcha, setShowCaptcha] = useState(true)

{showCaptcha && (
  <div className="fixed inset-0 z-50 flex items-center justify-center bg-black/50 backdrop-blur-sm">
    <div className="glass-modal w-96 p-6 rounded-xl">
      {/* 验证码图片占位 + 输入框 + 操作按钮 */}
    </div>
  </div>
)}
```

### 6.2 下载队列页（DownloadQueue）

**三种任务状态的视觉差异化：**

| 状态 | 实现方式 |
|------|---------|
| 活跃中 | 正常样式，进度条渐变动画，当前集 `animate-pulse` |
| 已暂停 | `opacity-60 grayscale` 降低视觉权重 |
| 错误 | `border-l-4 border-error/50`，显示失败信息和重试次数 |

TopBar 右侧速度指示器通过 `rightSlot` prop 传入。

### 6.3 动漫信息页（AnimeInfo）

- 海报列（`col-span-4`）：`position: sticky top-16`，浮动评分卡叠加
- 信息列（`col-span-8`）：超大字号标题（`text-7xl`），末词高亮主色
- 制作人员：2×2 bento 网格布局
- 元数据条：数据库 ID / 文件密度 / 元数据同步 / 导出记录

### 6.4 Biu 同步页（BiuSync）

- 左 7 列 / 右 5 列网格
- 推送（Push）和拉取（Pull）卡片用不同主色区分（primary / secondary）
- 同步状态面板：双色调渐变进度条

### 7.2 图片占位策略

Electron 沙箱环境无法加载任意外部图片 URL（如 Google 托管的演示图片）。所有设计稿中的图片替换为**样式化占位 div**：

```tsx

  <span className="material-symbols-outlined text-4xl text-outline">image</span>

```

## 八、开发与构建命令

```bash
cd front-end

# 开发模式（热更新）
npm run dev

# 生产构建
npm run build
# 输出：out/main/index.js, out/preload/index.js, out/renderer/
```

## 九、已知问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `Cannot find name '__dirname'` | tsconfig.node.json 缺少 Node 类型 | 添加 `"types": ["node"]` |
| `Parameter 'details' implicitly has an 'any' type` | TS strict 模式 | 显式标注 `(details: { url: string })` |
| IDE 报 "Cannot find module" | node_modules 未安装或 IDE 未索引 | 运行 `npm install`，重启 IDE |
| GUI 在 CI/沙箱环境无法启动 | 无显示器/GUI 运行时 | 仅在本地终端运行 `npm run dev`，不在自动化脚本中测试 |

## 十、后续：后端对接规划

当前所有页面均为静态 UI，无真实数据。后续对接路径：

1. **IPC 通道设计**：在 `preload/index.ts` 中通过 `contextBridge` + `ipcRenderer.invoke` 封装各功能接口
2. **主进程 IPC 处理**：在 `main/index.ts` 中 `ipcMain.handle` 注册处理函数，调用 Python 子进程（`child_process.spawn`）或直接调用 Node.js 实现
3. **Python 子进程通信**：通过 stdin/stdout JSON 流 或 本地 HTTP 服务（`localhost` 端口）与现有 Python 脚本通信

各页面对应的后端能力：

| 页面 | 对接脚本 |
|------|---------|
| Search & Download | `fetch_page.py` → `parse_anime_list.py` → `parse_watch_page.py` + `xifan_crawler.py` |
| Download Queue | `xifan_crawler.py` / `girigiri_download.py` 进度事件流 |
| Anime Info | `search_anime.py`（bgm.tv） |
| Biu Sync | `sync_biu.py` / `push_biu.py` |
