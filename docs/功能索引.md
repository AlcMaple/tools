## 架构速查

```
渲染进程 (React)                    主进程 (Node)
┌──────────────────┐                ┌──────────────────┐
│ pages/*.tsx      │   IPC invoke   │ ipc/*.ts         │
│ stores/*.ts      │ ─────────────▶ │   ↓              │
│ utils/*.ts       │ ◀───── event   │ bgm/xifan/…      │
│ components/*.tsx │                │ library/…        │
└──────────────────┘                └──────────────────┘
        │                                    │
        ▼                                    ▼
   localStorage                     userData/ 下的 JSON
   （渲染侧持久化）                  （主进程持久化）
```

- 渲染进程通过 `window.xxxApi.*` 调用主进程，API 定义在 [`src/preload/index.ts`](../src/preload/index.ts)
- 主进程所有 IPC 处理器集中在 [`src/main/ipc/`](../src/main/ipc/)，在 [`src/main/ipc/index.ts`](../src/main/ipc/index.ts) 统一注册
- 路由在 [`src/renderer/src/App.tsx`](../src/renderer/src/App.tsx)

## 3. 下载队列

统一展示所有下载任务，支持暂停、继续、单集重试、移除。

| 层 | 文件 |
|---|---|
| 页面 | [`src/renderer/src/pages/DownloadQueue.tsx`](../src/renderer/src/pages/DownloadQueue.tsx) |
| 渲染侧 store | [`src/renderer/src/stores/downloadStore.ts`](../src/renderer/src/stores/downloadStore.ts) |
| IPC 注册（分站点） | [`src/main/ipc/xifan.ts`](../src/main/ipc/xifan.ts)、[`src/main/ipc/girigiri.ts`](../src/main/ipc/girigiri.ts) |
| 主进程队列状态 | 在 `ipc/xifan.ts`、`ipc/girigiri.ts` 里用 `Map` 维护（in-memory） |

**队列设计（主进程）**
- 每个任务一个 queue：`pending[]` + `priorityFront[]` + `pausedEps: Set` + `AbortController`
- 单任务内**顺序下载**一集一集跑
- App 重启后主进程队列清空 → 渲染进程用 localStorage `xifan_download_tasks_v1` 恢复 → 通过 `*:download-resume` IPC 重建

**进度事件**
主进程通过 `xifan:download-progress` / `girigiri:download-progress` 广播事件；[`src/renderer/src/App.tsx`](../src/renderer/src/App.tsx) 顶层 `DownloadProgressListener` 挂载监听器，转发到 `downloadStore.handleProgressEvent`。

**导航保护**
[`src/renderer/src/utils/navGuard.ts`](../src/renderer/src/utils/navGuard.ts) 在 Settings 页脏数据时拦截导航，其他页面一般无需关心。

| 层 | 文件 |
|---|---|
| 页面 | [`src/renderer/src/pages/LocalLibrary.tsx`](../src/renderer/src/pages/LocalLibrary.tsx) |
| 封面组件 | [`src/renderer/src/components/CoverImage.tsx`](../src/renderer/src/components/CoverImage.tsx) |
| IPC 注册 | [`src/main/ipc/library.ts`](../src/main/ipc/library.ts) |
| 扫描 / 缩略图 / 监听 | [`src/main/library/api.ts`](../src/main/library/api.ts) |
| 扫描启动（启动时自动） | [`src/main/index.ts`](../src/main/index.ts)（`startLibraryWatch`、`reconcilePaths`） |
| Preload API | `window.libraryApi`（[`src/preload/index.ts`](../src/preload/index.ts)） |

**IPC 通道**
- `library:get-paths` / `library:add-path` / `library:remove-path`
- `library:get-entries` / `library:get-files` / `library:scan`
- `library:open-folder` / `library:play-video` / `library:play-folder`

**数据文件**（`userData/` 下）
- `library_paths.json` —— 用户配置的根目录
- `library_entries.json` —— 扫描结果缓存
- `thumbnails/` —— 抽取的缩略图（每次扫描清空重建）

**自定义协议**
主进程注册 `archivist://` 协议（[`src/main/index.ts:94`](../src/main/index.ts#L94)），渲染进程用 `archivist:///absolute/path.jpg` 加载本地缩略图。**注意**：Windows 盘符必须用三斜杠 `archivist:///C:/...`，否则 Chromium 会把 `C:` 解析成 host 导致盘符丢失。

## 7. 搜索结果缓存（跨模块）

独立一节，因为 Xifan / Girigiri / BGM 都用它。

| 文件 | 作用 |
|---|---|
| [`src/renderer/src/utils/searchCache.ts`](../src/renderer/src/utils/searchCache.ts) | 渲染侧 helper：`isSearchCacheEnabled` / `getCachedSearch` / `setCachedSearch` / `dedupRefresh` / TTL 常量 |
| [`src/main/ipc/system.ts`](../src/main/ipc/system.ts)（`cache:get`、`cache:set`） | 底层读写 `userData/search_cache.json` |

**缓存总开关**：`localStorage.xifan_settings.searchCacheEnabled`（Settings 页可切换）

**数据格式**（`search_cache.json`）
```jsonc
{
  "search_cache_xifan":    { "<关键词>": { "data": [...], "updatedAt": 1700000000000 } },
  "search_cache_girigiri": { "<关键词>": { "data": [...], "updatedAt": ... } },
  "search_cache_bgm":      { "<关键词>": { "data": [...], "updatedAt": ... } },
  "bgm_detail_cache":      { "<subjectId>": {...BgmDetail} },
  "archive_search_cache_xifan": { "<关键词>": [...XifanSearchResult] }
}
```

**TTL / 策略**（见 `TTL_BY_SOURCE`）
| 数据 | TTL | 过期行为 |
|---|---|---|
| Xifan 搜索卡片 | 30 天 | 当作 miss，走完整流程（可能触发验证码） |
| Girigiri 搜索卡片 | 30 天 | 同上 |
| BGM 搜索 | 14 天 | SWR：先显示旧结果，后台静默刷缓存（不动 UI），下次搜才看到新 |
| BGM 详情 | 永久 | 命中就用 |
| Xifan watch（集数列表） | **不缓存** | 每次实时拉，连载新集数必须可见 |

## 8. 错误展示

网络错误统一走 ErrorPanel，不用 `alert`。

| 文件 | 作用 |
|---|---|
| [`src/renderer/src/components/ErrorPanel.tsx`](../src/renderer/src/components/ErrorPanel.tsx) | UI 组件：图标 + 友好标题 + 提示 + Try again + Show details |
| [`src/renderer/src/utils/errorMessage.ts`](../src/renderer/src/utils/errorMessage.ts) | `friendlyError()`：根据原始错误判断类别（网络/HTTP 状态/解析/验证码），返回本地化文案 |

新接一处报错：把 `alert(...)` 换成 `setState({ status: 'error', message: String(err) })`，页面里渲染 `<ErrorPanel error={state.message} onRetry={...} />` 即可。

## 加新功能时的 checklist

1. **新 IPC 通道** → 见 `CLAUDE.md#adding-new-ipc-channels`
2. **新页面** → 在 `src/renderer/src/pages/` 新建 → 在 `App.tsx` 加 `<Route>` → 在 `components/Sidebar.tsx` 加入口
3. **新缓存项** → 优先复用 `searchCache.ts` 的结构（`{data, updatedAt}`），TTL 常量集中在 `TTL_BY_SOURCE`
4. **新报错路径** → 用 `ErrorPanel` + `friendlyError`，不用 `alert`
5. **需要本地文件展示** → 用 `archivist://` 协议，不要用 `file://`
